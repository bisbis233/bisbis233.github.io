服务器端

响应回到浏览器端

服务器端返回给浏览器端的东西

  • Access-Control-Allow-Methods 允许跨域的方法列表（GET、POST等）
  • Access-Control-Allow-Headers 允许跨域的Header列表
  • Access-Control-Expose-Headers 允许暴露给JavaScript代码的Header列表
  • Access-Control-Max-Age 最大的浏览器缓存时间，单位为s

声明提前

变量声明提前

var声明的变量在整个作用域可见，但var的赋值依照代码执行顺序执行

函数声明提前

函数声明也将在整个作用域可见

当用var声明函数时，由于赋值是按顺序的，因此x()执行的是一个undefined，会报错

变量提升优先级

函数声明的优先级更高，由于x先被声明为函数，在执行var x = 1时，会将x赋值为1。

let相关特性
  • 不允许在一个块级作用域重复定义（会报错）
  • 会变量提升，但提升形式不同（会报错）
  • 在全局作用域内声明变量，该变量如果与全局对象中的某属性同名，在用window.xxx时还是原属性
var相关特性
  • 定义的是函数作用域，在for(var j = 0; j < 3; j++) 中，j++是同步的，而循环体内是异步的，因此循环体内访问的i是循环最后的i
  • 在全局作用域内声明变量，该变量如果与全局对象中的某属性同名，在用window.xxx时是新声明的变量

作用域

当执行g()时，f()内的x是在定义f()的地方找，而不是在引用f()去寻找，这也称作静态作用域。

块级作用域

let与const所定义的变量适用于块级作用域，如let或const在某个函数内定义了x，y则在函数外是无法访问这两个变量的，即不会污染全局变量。

try...catch

上述代码try里面我们用到了一个没有申明的变量y，所以会报错，然后走到catch，catch会往作用域链最前面添加一个变量e，这是当前的错误对象，我们可以通过这个变量来访问到错误对象，这其实就相当于作用域链延长了。这个变量e会在catch块执行完后被销毁。

with语句可以操作作用域链，可以手动将某个对象添加到作用域链最前面，查找变量时，优先去这个对象查找，with块执行完后，作用域链会恢复到正常状态。

优点：内容区分为左中右三部分，其中左右两部分的宽度固定，中间部分的宽度随视口变化

做法：父元素 container 设置左右 padding 空出左右两栏位置。

三栏均设置向左浮动，中间栏设置 width: 100%; ，即与父元素宽度一样，将左右两栏挤到下面。

设置 margin 属性为负值将左右两栏移到上面，margin-left: -100%; 可以上移到上一行最左边， margin-left: -100px; 可以上移到上一行最右边（100px 是右边栏宽度）。

利用相对定位，将左右两栏定位到两边。

圣杯布局存在一定缺陷

  • 当某列内容溢出时，三栏不是等高的
  • 中间栏的最小宽度不能超过左右两栏的高度

在中间栏增加一个容器div把左右两侧空开

  • 双飞翼布局省去了很多 css，由于不使用定位，中间栏的最小宽度不再受限于左右两栏宽度。

缺点：

  • 双飞翼布局多加一层 dom 树节点，增加渲染树生成的计算量， dom 结构不如圣杯布局简单直观。
  • 三列等高问题仍然要单独解决。
